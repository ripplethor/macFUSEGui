macFUSE GUI for macOS

This is the dedicated macfuse gui page for macOS users who want SSHFS mounts without daily command-line mount workflows. macFUSEGui is a menu bar control layer on top of macFUSE + sshfs: it manages connection state, recovery, and usability.

What makes this different from CLI-only SSHFS

  • Independent per-remote connect and disconnect actions
  • Password and key authentication with Keychain-backed password storage
  • Auto-reconnect behavior after sleep/wake and network restoration
  • Clear status visibility from a native macOS menu bar app
  • Diagnostics snapshots for troubleshooting failed mounts

Install checklist

Use this checklist when evaluating or installing a macFUSE GUI on macOS:

  1. Install prerequisites: macFUSE and sshfs.
  2. Choose the correct build for your CPU architecture.
  3. Download the latest release and complete first-launch security approval.
  4. Add a remote host and test connection from the app UI.
brew install --cask macFUSE
brew install gromgit/fuse/sshfs-mac
uname -m

First launch and security note

Current public builds are unsigned and not notarized. Open once via Finder right-click (or Control-click) Open, or approve in System Settings → Privacy & Security.

Daily workflow

Typical usage is: create remote entry, connect, work through Finder/editor, and disconnect when done. If the network changes, recovery logic attempts to restore desired remotes automatically.

Troubleshooting quick checks

  • Verify both macFUSE and sshfs are installed.
  • Check that local mount paths do not collide with other remotes.
  • Confirm your credentials and host reachability.
  • Use diagnostics output to capture mount and recovery state.
